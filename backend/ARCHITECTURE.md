## ğŸ“ Key Design Decisions

### **1. Why UUIDs?**
- Avoids sequential ID leakage
- Enables distributed systems
- Standard for Supabase/PostgreSQL
- Future-proof for multi-region

### **2. Why Upsert?**
- APIs may return duplicate timestamps
- Handles late-arriving data
- Simplifies pipeline logic
- Idempotent operations

### **3. Why Separate Tables?**
- Normalized schema (3NF)
- Efficient querying
- Clear data ownership
- Easier to extend

### **4. Why Views?**
- Pre-computed joins
- Consistent query interface
- Query optimization
- Abstraction layer
